以下，再次总结下关于Git的知识。

Git

简介：

Git是一个分布式版本控制软件，最初由林纳斯·托瓦兹（Linus Torvalds）创作，于2005年以GPL发布。最初目的是为更好地管理Linux内核开发而设计。应注意的是，这与GNU Interactive Tools[5]（一个类似Norton Commander界面的文件管理器）有所不同。

git最初的开发动力来自于BitKeeper和Monotone[6][7]。git最初只是作为一个可以被其他前端（比如Cogito或Stgit[8]）包装的后端而开发的，但后来git内核已经成熟到可以独立地用作版本控制[9]。很多著名的软件都使用git进行版本控制[10]，中包括Linux内核、X.Org服务器和OLPC内核等项目的开发流程[11]。

历史：

自2002年开始，林纳斯·托瓦兹决定使用BitKeeper作为Linux内核主要的版本控制系统，以此来维护代码。因为BitKeeper为专有软件，林纳斯·托瓦兹决定使用BitKeeper来管理Linux内核源代码，这个决定长期在社区中遭受质疑。

2005年，BitKeeper的著作权拥有者，Larry MaVoy，宣称安德鲁·垂鸠曾试着以逆向工程手法来解析BitKeeper内部使用的协议，因此决定收回使用BitKeeper的授权。Linux内核开发团队与BitMover公司进行蹉商，但无法解决他们之间的歧见，林纳斯·托瓦兹决定自行开发版本控制系统，来替代BitKeeper，于是编写出git[16]。

在Linux社区中，特别是自由软件基金会的成员，主张应该使用开放源代码的软件来作为Linux核心的版本控制系统。林纳斯·托瓦兹曾经考虑过采用现成的软件来当成版本控制系统，例如Monotone，但这些现成的软件都存在一些问题，特别是性能不佳。

林纳斯·托瓦兹批评了CVS的架构。

功能：

git是用于Linux内核开发的版本控制工具。与CVS、Subversion一类的集中式版本控制工具不同，它采用了分布式版本库的作法，不需要服务器端软件，就可以运作版本控制，使得源代码的发布和交流极其方便。git的速度很快，这对于诸如Linux内核这样的大项目来说自然很重要。git最为出色的是它的合并追踪（merge tracing）能力。

实际上内核开发团队决定开始开发和使用git来作为内核开发的版本控制系统的时候，世界上开源社区的反对声音不少，最大的理由是git太艰涩难懂，从git的内部工作机制来说，的确是这样。但是随着开发的深入，git的正常使用都由一些友善的命令稿来执行，使git变得非常好用。现在，越来越多的著名项目采用git来管理项目开发，例如：wine、U-boot等[17]。

作为开源自由原教旨主义项目，git没有对版本库的浏览和修改做任何的权限限制，通过其他工具也可以达到有限的权限控制，比如：gitosis、CodeBeamer MR。原本git的使用范围只适用于Linux/Unix平台，但在Windows平台下的使用也日渐成熟，这主要归功于Cygwin、msysgit环境，以及TortoiseGit这样易用的GUI工具。git的源代码中也已经加入了对Cygwin与MinGW编译环境的支持且逐渐完善，为Windows用户带来福音。

用法

安装完Git后，在所需要上传代码的文件打开git bash，输入

git init          //初始化仓库

生成的.git文件用来记录上传文件的信息，版本，配置等。

准备好代码文件时准备上传时，输入

git add .        //将所有文件提交到环境中本地仓库

注意这里面是本地仓库，那怎么看是否上传到代码托管网站了呢？

git status       //查看代码状态以及是否上传等

如果上传了多次，怎么区分每一次上传代码的作用呢？

git commit -m "添加本次上传描述"     //添加描述

好了，已经准备好了一切，接下来准备上传

git push         //上传代码，需要输入托管仓库的邮箱及密码

如何搭建自己的『梯子』

Posted on 2016-02-19   |  

鉴于国内恶劣的『局域网』环境，想要获取相对精确的信息以及使用谷哥的特殊服务😜，尤为困难。因此拥有一把”梯子”显得非常重要。

下面列出常用的几种方法：

网络代理（Proxy）：一般常用的浏览器（Chrome，Firefox）即可实现，方便快捷；但免费服务不稳定，且经常有各种各样广告。

虚拟专用网（VPN）：免费VPN，服务不稳定且限制流量；曾使用过评价较好的VPN，但效果不尽人意。且未知服务商资质，通讯大多未用HTTPS加密，极易被服务提供商及第三方monitor。

那么，问题来了？啥都不能用，还怎么搞？

——哼 ฅ^._.^ฅ接下来就要由自己来搭建这些东西，这样就不用担心上面的问题咯。(● ω ●)

我们要实现的是使用VPS作为通讯的中转，然而在服务端和客户端部署服务，这里最主要的是服务端部署，客户端安卓可在谷哥Play下载Shadowsocks即可。




一丶前期准备

需要购买一台拥有root权限的VPS，那么应该选择那个提供商呢？首先，List自己的条件：1.国外服务提供商（国内只想呵呵）2.稳定（即使用人数较多）3.支持支付宝（关键劳资没有信用卡啊啊啊😂）
综上所述，其实选择不多了，这个我是用的搬瓦工https://bandwagonhost.com，除了以上优点外，还有最重要的一点是—–便宜”,,Ծ‸Ծ,,”。 （对了，说到这个，必须说一声某些人还是会玩，这个网站虽然能打开，但注册时的验证码被屏蔽了，我tm当时在一大堆英文里面找了好久。所以…）

购买后，搬瓦工会提供给你VPS主机地址，SSH端口（通过注册时填的邮箱）然后使用SSH工具登陆（Linux直接在终端使用SSH命令，Windows我使用的是开源的Putty），但在这之前需要通过搬瓦工官网进入控制面板设置root密码。

Linux下登陆ssh命令：

ssh -p vps 端口号 root@vpsIP地址

二丶搭建Shadowsock服务

选择的是CentOs，所以无法在使用sudo apt-get install安装了，不过这系列有自己的yum的什么鬼

2.1 安装组件

$ yum install m2crpto python-setuptools
$ easy_install pip
$ pip install shadowsocks

注意安装时要使用Y进行确认。

2.2 安装完成后配置服务器参数

$ vi  /etc/shadowsocks.json

写入如下配置

{
    "serve": "0.0.0.0",
    "server_port": 8388,   #服务端口，可自行更改
    "local_address": "127.0.0.1",
    "local_port": 1080,
    "password": "自己设置",
    "timeout": 300,
    "method": "aes-256-cfb",
    "fast_open": false,
    "workers:" 1
}

2.3 运行命令，启动Shadowsocks服务
运行下列命令

$ ssserver -c /etc/shadowsocks.json

至此，Shadowsocks服务端搭建完成，接下来客户端

Windows可在http://sourceforge.net/projects/shadowsocksgui/files/dist/,获取Shaowsocks客户端，配置好后，在所使用浏览器设置代理服务（Address：127.0.0.1 Port：1080）

Android端则在谷哥商店查找Shadowsocks即可

ios端在此不予赘述（劳资TM还用不起苹果），可参考http://letchinese.com/2015/04/12/build-your-own-vps/。
